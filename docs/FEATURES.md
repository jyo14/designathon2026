# Features

## Feature 1: Capture & Auto-Categorization

### What the user does
Pastes content into a single input field. Content can be:
- Plain text (a thought, a quote, a decision)
- A URL (article, Figma frame, tweet, blog post)
- A screenshot (uploaded image)
- A combination (text + URL together)

Optional: adds a one-line note alongside ("for JARVIS case study" / "look at this later").

### What the system does
Sends the capture to Gemini API for classification. Returns:
- **Label** — one of: `UI Pattern`, `Case Study Fragment`, `Research`, `Inspiration`, `Project Decision`, `To-Do for Portfolio`
- **Summary** — 1–2 lines
- **Themes** — 2–4 short tags (e.g., `conversation-design`, `slot-filling`, `multilingual-ux`)
- **Project link** — if the capture clearly relates to a known project, link it

Captures are stored in localStorage as a list, newest first.

### UI requirements
- Single capture form on the home page (collapsible — should not dominate the screen)
- Each capture displays: timestamp, label (as a colored chip), summary, themes, source URL/image if any
- Synthesizing state shown while Gemini processes (~2–3 seconds)

## Feature 2: Daily Brief

### What it is
The home screen of the app. Not a real notification — an in-app view that's the first thing the user sees when they open the app each day.

### Content (generated by AI, regenerable on demand)

1. **Today's Top 3** — three items to focus on, each with reasoning. Items can be:
   - A capture that's been sitting unopened (e.g., "you saved this UI pattern 6 days ago and your active project needs it")
   - A case study section that's stalled
   - A research thread you've returned to repeatedly

2. **2 Connections Worth Noticing** — pairings of captures the system thinks belong together (e.g., "the slot-filling article from Tuesday connects to your JARVIS error-state decision from Monday")

3. **1 Nudge** — overdue or quietly-stalled item ("you saved 4 articles on multilingual UX last week and opened none of them" / "JARVIS case study Section 3 hasn't been touched in 8 days")

### Constraints
- No inbox content. No calendar content. No meeting prep. The brief is purely about the user's saved material and project work.
- Brief regenerates on demand (button), not on a schedule (no cron jobs).
- Brief is timestamped so the user can see when it was last generated.

## Feature 3: Portfolio Structure Mapping

### What it is
Given a project (e.g., "JARVIS"), the system maps the user's captures into a case study structural template, suggesting which captures belong in which section. It does NOT write prose.

### The single template (for prototype)
**Case Study: Problem → Process → Outcome**
- Section 1: The Problem (who, what, why)
- Section 2: Discovery & Research (what we learned)
- Section 3: Process (decisions made, alternatives considered, trade-offs)
- Section 4: The Solution (what we built and why)
- Section 5: Outcome & Reflection (results, learnings, what's next)

### What the system does
For a selected project, takes all captures tagged with that project, sends to Gemini API. Returns:
- A suggested mapping: which captures fit into which section
- Gaps: sections with no captures yet
- One-line suggestions for the gaps (e.g., "Section 5 has no outcome material — consider documenting a user-test observation")

### What the system does NOT do
- It does not write the case study itself
- It does not fill in section content
- It does not generate prose anywhere

A visible label on the page: **AI structures. You write.**
